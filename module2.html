Lesson 2.1 ‚Äî How Networks Work

A network is simply two or more devices connected together to share data and resources. Understanding networks is essential for cybersecurity ‚Äî almost all modern attacks happen over networks.

Types of Networks

LAN ‚Äî Local Area Network

Covers a small area like a home, office, or building. Your home Wi-Fi is a LAN. All devices connect to a router/switch.

WAN ‚Äî Wide Area Network

Covers large geographic areas. The internet is the largest WAN. Connects multiple LANs together across cities or countries.

MAN ‚Äî Metropolitan Area Network

Covers a city or metropolitan area. Often used by ISPs and municipalities.

WLAN ‚Äî Wireless LAN

A LAN that uses Wi-Fi instead of cables. Introduces additional wireless security considerations.

Key Network Devices

üîÄ
Router
Connects different networks together and directs (routes) traffic between them. Your home router connects your LAN to the internet (WAN). Makes routing decisions based on IP addresses.
üîå
Switch
Connects devices within the same LAN. Forwards data based on MAC addresses (physical hardware addresses). More secure and efficient than older hubs.
üî•
Firewall
Monitors and filters network traffic based on predefined rules. Can be hardware or software. Acts as a gatekeeper between trusted and untrusted networks.
üì°
Access Point (AP)
Provides wireless connectivity to a wired network. Creates Wi-Fi coverage. Rogue access points (fake Wi-Fi) are a major security concern.

üí° Security Insight

Most cyber attacks travel over networks. Understanding how data flows helps you understand how attacks happen and where to place defenses.

Lesson 2.2 ‚Äî IP Addresses

Every device on a network needs an IP (Internet Protocol) address ‚Äî like a home address for data to know where to be delivered.

IPv4 vs IPv6

IPv4
192.168.1.1
4 numbers separated by dots, each 0-255. About 4.3 billion addresses ‚Äî we've run out! Still the most common format.
IPv6
2001:0db8::8a2e:7334
8 groups of hex digits. Practically unlimited addresses. Slowly replacing IPv4 as the internet grows.

DHCP vs Static IP

DHCP (Dynamic)

Server automatically assigns IP addresses. Most home and office devices use this. The IP may change over time.

Static IP

Manually configured, never changes. Used for servers, printers, and network devices that need a consistent address.

üìå Security Note

IP addresses can be spoofed ‚Äî attackers can fake their IP to hide their identity or bypass IP-based filtering. IP addresses alone are not reliable for authentication.

Lesson 2.3 ‚Äî DNS (Domain Name System)

DNS is the internet's phone book. Humans remember names (google.com), computers use numbers (142.250.80.46). DNS translates between the two.

How DNS Works

1
You type google.com in your browser
2
Your device checks its local DNS cache ‚Äî is the answer already stored?
3
If not, it asks the DNS resolver (usually your ISP or Google 8.8.8.8)
4
The resolver queries root servers ‚Üí TLD servers ‚Üí authoritative DNS server
5
The IP address 142.250.80.46 is returned and your browser connects

DNS Attacks You Must Know

DNS Poisoning (Cache Poisoning)

Attacker injects false DNS records into a resolver's cache. Users get redirected to a malicious IP even when typing a legitimate URL. Used for phishing and malware distribution.

DNS Hijacking

Attacker takes control of DNS records (often by compromising the domain registrar). All traffic for a domain is redirected to attacker-controlled servers.

DNS Tunneling

Attackers encode malicious data in DNS queries to exfiltrate data or communicate with malware. Often bypasses firewalls since DNS traffic is usually allowed.

üõ°Ô∏è Defense: DNSSEC

DNSSEC (DNS Security Extensions) adds digital signatures to DNS records, allowing resolvers to verify that responses are authentic and haven't been tampered with.

What is TLS?

TLS (Transport Layer Security) is the encryption protocol that makes HTTPS secure. It replaced the older, insecure SSL protocol.

‚ùå SSL‚Äî Deprecated, multiple critical vulnerabilities (POODLE, BEAST, HEARTBLEED)
‚ö†Ô∏è TLS 1.0/1.1‚Äî Old versions, deprecated by most browsers
‚úÖ TLS 1.2‚Äî Widely supported, still acceptable
‚úÖ TLS 1.3‚Äî Current standard, fastest and most secure

üîç The Padlock Icon

The padlock in your browser means the connection is encrypted via TLS. However, it does NOT mean the website itself is trustworthy ‚Äî phishing sites can also have HTTPS. Always verify the domain name too.
