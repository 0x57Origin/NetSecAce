Lesson 3.1 ‚Äî Malware Types

Malware (malicious software) is any software intentionally designed to cause harm. You must know every type for the exam.

Virus Attaches to files

A piece of code that attaches itself to legitimate files and spreads when those files are opened. Requires user action to spread.

Real example: ILOVEYOU (2000) ‚Äî spread via email attachment, infected 10+ million computers
Worm Self-replicates

Spreads across networks automatically without any user action. Exploits network vulnerabilities to copy itself to other systems.

Real example: WannaCry (2017) ‚Äî used EternalBlue exploit to spread to 200,000+ computers in 150 countries
Trojan (Trojan Horse) Disguised as legit software

Disguises itself as legitimate, useful software to trick users into installing it. Does not replicate ‚Äî relies on social engineering.

Real example: Fake antivirus programs, cracked software downloads containing malware
Ransomware Encrypts files

Encrypts the victim's files and demands payment (usually cryptocurrency) for the decryption key. Attacks availability and potentially confidentiality.

Real example: LockBit, REvil, WannaCry ‚Äî have caused billions in damages to hospitals, governments, and businesses
Spyware Silent monitoring

Silently monitors user activity and steals sensitive information like keystrokes, passwords, screenshots, and browsing history.

Includes keyloggers ‚Äî records everything you type. Used in financial fraud and corporate espionage.
Rootkit Hides deep in OS

Embeds itself deep in the operating system (often at kernel level) to conceal its presence and maintain persistent stealthy access. Very hard to detect and remove.

Often requires a complete OS reinstall to remove. Hides other malware from antivirus tools.
Botnet Network of infected devices

A network of compromised devices (bots) controlled remotely by an attacker (botmaster) without the owners' knowledge. Used for DDoS attacks, spam, and crypto mining.

Real example: Mirai botnet (2016) ‚Äî infected 600,000+ IoT devices, launched massive DDoS attacks
Adware Unwanted ads

Displays unwanted advertisements. Often bundled with free software. Usually not directly harmful but privacy-invasive ‚Äî can slow systems and redirect browsers.

Example: Browser toolbars, pop-up ad generators, search engine hijackers

Lesson 3.2 ‚Äî Social Engineering

Social engineering attacks manipulate human psychology rather than exploiting technical vulnerabilities.

Humans are consistently the weakest link in security. No firewall can stop an employee from being tricked.

Phishing Variants

Phishing

Mass emails pretending to be a trusted source (bank, Netflix, Amazon). Casts a wide net ‚Äî even a 1% success rate yields thousands of victims.

Spear Phishing

Targeted phishing against a specific individual. Attacker researches the victim ‚Äî uses their name, job title, colleague names. Much more convincing.

Whaling

Spear phishing targeting high-value individuals ‚Äî CEOs, CFOs, board members. "Going after the big fish." Often involves fake wire transfer requests.

Vishing

Voice phishing ‚Äî phone calls. Attacker impersonates bank, IRS, or tech support. Uses caller ID spoofing to appear legitimate.

Smishing

SMS phishing ‚Äî text messages. "Your package delivery failed, click here." Or fake bank fraud alerts with malicious links.

Pharming

Redirects users to a fake website even when they type the correct URL. Often done via DNS poisoning or hosts file modification.

Lesson 3.3 ‚Äî Password Attacks

Passwords are the most common authentication method ‚Äî and the most frequently attacked. Know every method.

Brute Force Attack

Systematically tries every possible combination of characters until the correct password is found. Time-consuming but guaranteed to work eventually.

aaa ‚Üí aab ‚Üí aac ‚Üí ... ‚Üí zzz ‚Üí aaa1 ‚Üí ...
Defense: Strong passwords, account lockout policies, MFA
Dictionary Attack

Uses a pre-built list of common words, passwords, and phrases to guess credentials. Much faster than brute force.

password ‚Üí password1 ‚Üí 123456 ‚Üí qwerty ‚Üí letmein ‚Üí ...
Defense: Passphrases, avoiding common words, password managers
Credential Stuffing

Uses username/password pairs leaked from previous data breaches to attempt logins on other websites. Works because users reuse passwords.

Defense: Unique passwords for every account (use a password manager!), MFA
Password Spraying

Tries one or a few common passwords against many accounts to avoid lockouts. Slower but stealthy ‚Äî avoids triggering account lockout policies.

user1:Password123! ‚Üí user2:Password123! ‚Üí user3:Password123! ‚Üí ...
Defense: MFA, monitoring for widespread failed login attempts
Rainbow Table Attack

Uses precomputed tables of hash values to reverse hashed passwords. Attacker steals the hashed password database, then looks up the hash in the table.

Defense: Salting passwords ‚Äî adds a unique random value before hashing, defeating precomputed tables
Keylogger

Software or hardware that records every keystroke typed by the victim. Captures passwords, credit card numbers, and messages.

Defense: Anti-malware software, virtual keyboards, hardware security keys

Lesson 3.4 ‚Äî Network Attacks

Man-in-the-Middle (MitM)

Attacker secretly positions themselves between two communicating parties, intercepting and potentially modifying data without either party knowing.

Real scenario: On a coffee shop Wi-Fi, attacker intercepts your unencrypted HTTP traffic. Defense: HTTPS, VPN
DDoS ‚Äî Distributed Denial of Service

Floods a target server with massive traffic from thousands of compromised devices (botnet), overwhelming it until it crashes and becomes unavailable.

Attacks Availability. Defense: CDNs, rate limiting, anti-DDoS services (Cloudflare)
Packet Sniffing

Capturing and reading network traffic data. On unencrypted networks, everything can be read in plain text ‚Äî passwords, emails, data.

Defense: Encrypt all traffic (HTTPS, TLS, VPN), avoid public Wi-Fi for sensitive activities
Spoofing

Faking identity to gain trust or bypass controls.

‚Ä¢ IP Spoofing ‚Äî fake source IP address
‚Ä¢ MAC Spoofing ‚Äî fake hardware address
‚Ä¢ Email Spoofing ‚Äî fake sender address (phishing)
‚Ä¢ Caller ID Spoofing ‚Äî fake phone number (vishing)
Replay Attack

Attacker captures a valid authentication transmission and re-sends it to gain unauthorized access. Like recording your keycard beep and playing it back later.

Defense: Timestamps, session tokens, nonces (numbers used once) in authentication protocols
ARP Poisoning

Sends fake ARP (Address Resolution Protocol) replies to associate the attacker's MAC address with a legitimate IP. Redirects network traffic through the attacker (enables MitM).

Defense: Dynamic ARP Inspection (DAI), static ARP entries for critical systems

Lesson 3.5 ‚Äî Application Attacks

SQL Injection

Inserts malicious SQL code into input fields to manipulate the backend database. Can bypass authentication, extract all data, or delete the entire database.

Login form: Username field
admin' OR '1'='1
Query becomes: SELECT * FROM users WHERE username='admin' OR '1'='1'
-- Always true! Bypass login.
Defense: Parameterized queries, input validation, least privilege database accounts
XSS ‚Äî Cross-Site Scripting

Injects malicious scripts into web pages that are then executed by other users' browsers. Can steal session cookies, redirect users, or deface websites.

Defense: Input sanitization, Content Security Policy (CSP), output encoding, HttpOnly cookies
Zero-Day Attack

Exploits a vulnerability that the software vendor doesn't know about yet ‚Äî zero days of warning or patching time. Extremely dangerous because no patch exists.

Defense: Defense in depth, behavioral monitoring, prompt patching of known vulnerabilities
Buffer Overflow

Sends more data than a program's buffer can hold, overwriting adjacent memory. Can crash programs or allow attackers to execute arbitrary code.

Defense: Input validation, memory-safe programming languages, ASLR (Address Space Layout Randomization)
CSRF ‚Äî Cross-Site Request Forgery

Tricks an authenticated user's browser into sending unauthorized requests to a website they're logged into. Example: visit a malicious page that secretly transfers money from your bank.

Defense: CSRF tokens, SameSite cookies, verifying request origin headers
Directory Traversal

Manipulates file path inputs to access files outside the intended directory. Can expose system files, configuration files, or other sensitive data.

../../etc/passwd
